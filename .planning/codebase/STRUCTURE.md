## Directory Purposes

**`src/app/`:**
- Purpose: Next.js App Router pages and layout
- Contains: Root layout, page components, global styles, metadata
- Key files: `layout.tsx` (persistent shell), `page.tsx` (home route), `globals.css` (theme)

**`src/components/`:**
- Purpose: Reusable React UI components
- Contains: Page sections, header/footer, modal, animations
- Key files: All files are core to the page; no unused components

**`src/lib/`:**
- Purpose: Shared utilities and helpers
- Contains: Styling utilities, functions
- Key files: `utils.ts` (classname merging via clsx + tailwind-merge)

**`public/`:**
- Purpose: Static assets served directly
- Contains: Logo images, SVG icons
- Key files: `Logos/t-che 2 as.png` (dark logo), `Logos/t-che 2-white.png` (white logo)

**`.planning/codebase/`:**
- Purpose: GSD mapping and planning documents
- Contains: Architecture, structure, conventions, testing, concerns analysis
- Key files: Markdown documents generated by GSD tools

## Key File Locations

**Entry Points:**
- `src/app/layout.tsx`: Root layout renders Header, main, Footer, ContactModal
- `src/app/page.tsx`: Home page composes all section components
- `next.config.ts`: Next.js configuration (image remotePatterns for Unsplash)

**Configuration:**
- `tsconfig.json`: TypeScript settings, path alias `@/*` → `./src/*`
- `package.json`: Dependencies (React 19, Next.js 16, Framer Motion, Tailwind, Lucide)
- `eslint.config.mjs`: ESLint rules (Next.js + TypeScript standards)
- `postcss.config.mjs`: PostCSS setup for Tailwind CSS v4

**Core Logic:**
- `src/components/Header.tsx`: Navigation, mobile menu, contact trigger
- `src/components/Hero.tsx`: Landing section with stats and CTA
- `src/components/Services.tsx`: Service cards with Framer Motion stagger
- `src/components/FeaturedProjects.tsx`: Project gallery with masonry layout
- `src/components/CompanyValues.tsx`: Company info, stats, core standards
- `src/components/Footer.tsx`: Contact info, social links, footer navigation
- `src/components/ContactModal.tsx`: Event-driven modal for contact display

**Styling:**
- `src/app/globals.css`: Tailwind theme variables, custom backgrounds, root styles
- `src/lib/utils.ts`: `cn()` function for safe classname merging

**Static Assets:**
- `public/Logos/t-che 2 as.png`: Brand logo (dark)
- `public/Logos/t-che 2-white.png`: Brand logo (white/footer)

## Naming Conventions

**Files:**
- Components: PascalCase (e.g., `Header.tsx`, `FeaturedProjects.tsx`)
- Utilities: camelCase (e.g., `utils.ts`)
- Styles: kebab-case for CSS classes (e.g., `bg-primary`, `text-gray-500`)

**Directories:**
- Feature directories: lowercase (e.g., `components`, `app`, `lib`)
- Nested directories: lowercase (e.g., `Logos` inside `public`)

**TypeScript & Code:**
- Exported components: PascalCase function names
- Variables and functions: camelCase (e.g., `isMobileMenuOpen`, `openContact`)
- Constants (animation objects): camelCase (e.g., `containerVariants`, `itemVariants`)
- Types: Implicit via TypeScript inference; `React.ReactNode` for children

**CSS/Tailwind:**
- Utility classes: Tailwind standard format (e.g., `flex`, `gap-4`, `md:col-span-8`)
- Color tokens: `primary`, `accent`, `surface`, `text-main`, `text-muted`
- Custom backgrounds: `bg-blueprint`, `bg-grid-slate-900`, `bg-concrete`, `bg-dot-black`

## Where to Add New Code

**New Feature (e.g., Services Page, Blog):**
- Route file: `src/app/services/page.tsx` (App Router)
- Layout if needed: `src/app/services/layout.tsx`
- Associated components: `src/components/ServiceDetail.tsx`, etc.
- Styling: Inline Tailwind in components or extend `globals.css` if global

**New Section Component:**
- Implementation: `src/components/NewSection.tsx`
- Wrapper: "use client" at top
- Structure: motion.div with viewport animations, max-w-7xl container, semantic section
- Styling: Tailwind with custom theme colors from `globals.css`
- Animation variants: Define locally in component or import from shared pattern

**New Utility Function:**
- Location: `src/lib/utils.ts` (if general-purpose) or inline in component
- Pattern: Named export, typed with TypeScript, documented with JSDoc if complex
- Example: `export function formatPrice(amount: number): string { ... }`

**New Icon or Asset:**
- Icons: Import from lucide-react (`import { IconName } from 'lucide-react'`)
- Images: Place in `public/` subdirectory and reference with `next/image` and remote URL or relative path
- Logos: Place in `public/Logos/`

**Styling Extensions:**
- Global theme colors: Add to `@theme` block in `src/app/globals.css`
- Custom utilities: Use `@apply` in `globals.css` or define Tailwind config
- Per-component styles: Use Tailwind classes in JSX or inline styles

## Special Directories

**`.next/`:**
- Purpose: Next.js build output (compiled app, optimized assets)
- Generated: Yes (automatic during build)
- Committed: No (in `.gitignore`)

**`node_modules/`:**
- Purpose: npm dependencies
- Generated: Yes (npm install)
- Committed: No (in `.gitignore`); locked via package-lock.json

**`.git/`:**
- Purpose: Git repository metadata
- Generated: Yes (git init)
- Committed: Not version-controlled

**`public/Logos/`:**
- Purpose: Brand assets (company logos)
- Generated: No (manually added)
- Committed: Yes (tracked in git)

## File Operations Guide

**Modifying Styles:**
1. Update color theme: Edit `@theme` block in `src/app/globals.css`
2. Add custom backgrounds: Define as CSS classes in `globals.css` with `@apply` or raw CSS
3. Component styles: Use Tailwind classes directly in JSX via `className` prop

**Adding Pages:**
1. Create `src/app/[route]/page.tsx`
2. Export default component
3. Styling handled via Tailwind + `globals.css` theme
4. Navigation updated in Header links

**Modifying Components:**
1. Edit component file in `src/components/`
2. Update imports if dependencies change
3. Adjust Tailwind classes for styling
4. Test animations by viewing in browser

**Updating Theme Colors:**
1. Edit CSS variables in `src/app/globals.css` under `@theme` block
2. Change `--color-primary`, `--color-accent`, `--color-surface`, etc.
3. Classes using `bg-primary`, `text-primary` automatically update
4. No component changes needed
